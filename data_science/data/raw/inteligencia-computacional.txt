Que es Rest?,Representational State Transfer
Estilo arquitectonioc para disenar servicios web que se comunican mediante HTTP.
Principios de Rest,Cliente-servidor: Separacion de responsabilidaddes
Stateless: Cada peticion contiene informacion necesaria
Recursos: Recurso identificable mediante URLS.
Metodos HTTP: Uso semantico de GET, POST, PUT, DELETE

Patch - actualizar
Delete - eliminar recursos
Codigos respuesta HTTP,200 - OK
201 - Created
400 - Bad Request
404 - Not Found
500 - Internal Server Error
Rest para ML,Exponer un modelo de ML previamente entrenado para realizar predicciones en tiempo real.
Esto permite a aplicaciones cliente (web, móviles, etc.) consumir el modelo a través de HTTP.
Es especialmente útil para proveer inferencias como servicio o integrar modelos en sistemas másgrandes, sin necesidad de compartir el código del modelo o los datos de entrenamiento.
Flujo típico
1. Cliente envía datos de entrada (features) en una petición POST
2. Servidor procesa la petición con el modelo
3. Servidor responde con la predicción
