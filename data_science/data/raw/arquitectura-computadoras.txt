¿Qué es la Arquitectura de Von Neumann?,Modelo de diseño donde tanto los datos como las instrucciones del programa se almacenan en la misma memoria unificada, accediendo a ellos a través de un bus común.
Cuál es el "Cuello de botella de Von Neumann",La limitación de rendimiento que ocurre porque la CPU procesa información mucho más rápido de lo que el bus de memoria puede transferir datos e instrucciones.
Qué diferencia a la Arquitectura Harvard de la Von Neumann,La Arquitectura Harvard tiene memorias y buses físicamente separados para los datos y para las instrucciones, permitiendo lectura simultánea.
¿Cuál es la función de la ALU (Unidad Aritmético Lógica)?,Es el componente del procesador responsable de ejecutar todas las operaciones matemáticas (sumas, restas) y operaciones lógicas (AND, OR, NOT).
¿Qué es un Registro de CPU?,Es la memoria más pequeña, rápida y costosa del sistema. Se encuentra dentro del procesador y retiene los datos inmediatos con los que la ALU está operando.
Qué hace el Program Counter (Contador de Programa)?,Es un registro especial que almacena la dirección de memoria de la siguiente instrucción que la CPU debe ejecutar.
Describe el ciclo de instrucción básico de una CPU (Fetch-Decode-Execute),1. Fetch: Trae la instrucción de la memoria. 
2. Decode: Interpreta qué operación es. 
3. Execute: La ALU ejecuta la lógica y guarda el resultado.
¿Qué es la Memoria Caché y por qué existe?,Es una memoria volátil de alta velocidad ubicada entre la CPU y la RAM. Existe para mitigar la diferencia de velocidad entre el procesador y la memoria principal, almacenando datos de acceso frecuente.
Explica la jerarquía de la Caché (L1, L2, L3),L1 es la más pequeña y rápida, exclusiva de cada núcleo. L2 es más grande y un poco más lenta (por núcleo). L3 es la más grande, lenta y compartida entre todos los núcleos del procesador.
¿Qué es un "Cache Miss"?,Ocurre cuando la CPU busca un dato en la memoria caché y no lo encuentra, viéndose forzada a ir a buscarlo a una memoria más lenta (caché de nivel inferior o RAM).
¿Qué es el Pipelining (Segmentación de cauce)?,Arquitectura que permite a la CPU solapar la ejecución de múltiples instrucciones. Mientras una instrucción se está ejecutando, la siguiente ya se está decodificando, aumentando el "throughput".
¿Qué es la Arquitectura RISC (Reduced Instruction Set Computer)?,Filosofía de diseño de procesadores (como ARM) que utiliza un conjunto pequeño y altamente optimizado de instrucciones simples, ejecutadas en un solo ciclo de reloj.
¿Qué es la Arquitectura CISC (Complex Instruction Set Computer)?,Filosofía de diseño (como x86) que utiliza instrucciones complejas, donde una sola instrucción puede ejecutar múltiples operaciones de bajo nivel (ej. leer memoria, sumar y guardar).
Qué es un Interrupción (Interrupt) a nivel de hardware,Una señal enviada a la CPU por un dispositivo (ej. teclado, disco) indicando que requiere atención inmediata, pausando el proceso actual.
Qué es DMA (Direct Memory Access)?,Subsistema que permite a los periféricos transferir datos directamente a la RAM sin intervención continua de la CPU, liberando al procesador para otras tareas.
¿Cuál es la función principal del Kernel (Núcleo)?,Es el componente central del Sistema Operativo que gestiona los recursos del hardware y sirve de puente entre el software (aplicaciones) y el hardware físico (CPU, memoria, dispositivos).
¿Qué diferencia hay entre el "User Mode" y el "Kernel Mode"?,En User Mode, el software tiene acceso restringido y no puede tocar hardware directamente. En Kernel Mode, el código tiene acceso irrestricto y total a los recursos del sistema.
Qué es una System Call (Llamada al Sistema)?,Es la interfaz o mecanismo lógico mediante el cual un programa en User Mode solicita un servicio privilegiado al Sistema Operativo (como leer un archivo o reservar memoria), forzando un cambio temporal a Kernel Mode.
Qué es la Paginación (Paging) en memoria?,Esquema de gestión donde la memoria física y lógica se dividen en bloques del mismo tamaño (páginas y marcos). Permite que la memoria física de un proceso no sea contigua.
Qué es la Segmentación en memoria?,División lógica de la memoria en bloques de tamaño variable, basados en la estructura del programa (ej. un segmento para el código fuente, otro para variables globales, otro para la pila).
Qué es la Memoria Virtual?,Abstracción arquitectónica que engaña a los procesos haciéndoles creer que tienen una memoria contigua y enorme, mapeando direcciones lógicas a RAM física o a un espacio temporal en el disco duro.
¿Qué es el "Swapping" (Intercambio)?,El proceso por el cual el SO mueve temporalmente páginas completas de memoria de un proceso inactivo desde la RAM al disco duro (Swap space) para liberar RAM física.
¿Qué es el TLB (Translation Lookaside Buffer)?,Es una pequeña y ultrarrápida memoria caché de hardware dentro de la MMU (Memory Management Unit) que guarda las traducciones recientes de direcciones de memoria lógica a física, acelerando la paginación.
¿Qué es un "Page Fault" (Fallo de Página)?,Excepción de hardware que ocurre cuando un proceso intenta acceder a una página de memoria que está mapeada en su espacio virtual pero actualmente no está cargada en la RAM física (generalmente está en el disco).
¿Qué es el Thrashing (Hiperpaginación)?,Colapso del rendimiento donde el sistema pasa más tiempo intercambiando páginas entre la RAM y el disco (Swapping) que ejecutando instrucciones reales, causado por falta de memoria RAM física.
Estructura de memoria: ¿Qué almacena el "Stack" (Pila)?,Almacena variables locales, parámetros de funciones y el flujo de control (direcciones de retorno) con un tamaño y ciclo de vida predecible. Es muy rápido.
Estructura de memoria: ¿Qué almacena el "Heap" (Montículo)?,Almacena objetos y estructuras de datos dinámicos cuyo tamaño no se conoce en tiempo de compilación. Requiere asignación manual (o Garbage Collector) y es más lento que el Stack.
Describe el algoritmo de reemplazo de páginas LRU,Least Recently Used (Menos Usado Recientemente). Cuando la RAM está llena, el SO expulsa la página de memoria que lleva más tiempo sin ser accedida.
¿Qué es el MBR (Master Boot Record)?,Es el primer sector del disco duro que contiene la tabla de particiones y el pequeño código inicial (Bootloader) necesario para cargar el Sistema Operativo en RAM.
¿Cuál es la diferencia entre los sistemas de archivos FAT32 y NTFS/ext4?,FAT32 es antiguo, simple y universal pero no admite archivos mayores a 4GB. NTFS (Windows) y ext4 (Linux) son transaccionales (Journaling), admiten metadatos avanzados, permisos y archivos masivos.
Define qué es un Proceso,Es un programa en ejecución. Tiene su propio espacio de memoria aislado (Stack, Heap, Data, Text) y recursos asignados por el Sistema Operativo.
Define qué es un Hilo (Thread),Es la unidad más pequeña de ejecución dentro de un proceso. Múltiples hilos dentro del mismo proceso comparten el mismo Heap y espacio de direcciones, pero tienen su propio Stack y Program Counter.
¿Qué es el PCB (Process Control Block)?,Una estructura de datos en el Kernel que guarda todo el estado de un proceso (ID, registros de CPU, estado actual, punteros de memoria) para poder pausarlo y reanudarlo sin perder información.
Qué es un Context Switch (Cambio de Contexto)?,La acción lógica donde el SO detiene la ejecución de un proceso/hilo, guarda su estado actual en el PCB/TCB, y carga el estado de un proceso diferente para que la CPU lo ejecute. Es una operación costosa.
Enumera los 5 estados principales del ciclo de vida de un proceso,Nuevo (New), Listo (Ready), En ejecución (Running), Bloqueado/En espera (Blocked/Waiting), y Terminado (Terminated).
¿Qué significa que un proceso esté en estado "Bloqueado" (Blocked)?,Significa que el proceso no puede continuar ejecutándose porque está esperando un evento externo (ej. la respuesta de un disco duro, un input del usuario, o un mensaje de red).
¿Qué es el "Scheduler" (Planificador)?,Es el componente del SO que implementa algoritmos lógicos para decidir qué proceso en el estado "Listo" (Ready) debe obtener el control de la CPU en el siguiente ciclo.
Algoritmo de planificación: ¿Qué es Round Robin?,Algoritmo preventivo donde a cada proceso se le asigna un intervalo de tiempo fijo ("Quantum") en la CPU. Si no termina en ese tiempo, es interrumpido y mandado al final de la fila.
Algoritmo de planificación: ¿Qué es FCFS (First-Come, First-Served)?,Algoritmo no preventivo donde el primer proceso que llega a la cola de "Listo" es el primero en ser ejecutado hasta que termine o se bloquee. Puede causar el "Efecto Convoy".
¿Qué es una "Race Condition" (Condición de Carrera),Un fallo de diseño concurrente donde el resultado final de un sistema depende del orden temporal impredecible en que se ejecutan múltiples hilos que acceden a una memoria compartida.
¿Qué es un Mutex (Mutual Exclusion)?,Un mecanismo de sincronización lógico (un "candado"). Solo un hilo puede poseer el Mutex a la vez para acceder a una Sección Crítica; los demás hilos deben esperar a que se libere.
¿Qué diferencia hay entre un Semáforo y un Mutex?,Un Mutex permite el acceso a solo un hilo a la vez (binario). Un Semáforo tiene un contador lógico que permite el acceso a N cantidad de hilos simultáneamente (ej. limitar una conexión a 5 hilos máximo).
¿Cuáles son las 4 condiciones necesarias de Coffman para que ocurra un Deadlock?,1. Exclusión mutua, 2. Retener y esperar, 3. No apropiación (No preemption), 4. Espera circular.
¿Qué es la "Sección Crítica" en programación concurrente?,Es el segmento de código arquitectónico donde un proceso o hilo accede y modifica recursos compartidos (variables, archivos) que requieren protección para evitar condiciones de carrera.
¿Qué es un proceso Huérfano (Orphan) y un proceso Zombie?,Huérfano: Su proceso padre terminó o murió antes que él (es adoptado por el proceso inicial init)
 Zombie: Ya terminó su ejecución, pero su entrada en la tabla de procesos sigue existiendo porque su padre no ha leído su estado de salida.
Aumentar la memoria Caché L1 infinitamente hará que el procesador sea infinitamente rápido.,Falso
 A mayor tamaño de caché, mayor es la latencia y el tiempo necesario para buscar un dato dentro de ella (se vuelve más lenta lógicamente).
Los hilos dentro de un mismo proceso no pueden comunicarse entre sí sin intervención del Kernel,Falso
Al compartir el mismo espacio de memoria (Heap), pueden leer y escribir las mismas variables globales directamente, lo que los hace rápidos pero peligrosos (requieren Mutex).
Una llamada al sistema (System Call) consume más ciclos de reloj que la ejecución de una función normal,Verdadero
 Requiere un cambio de contexto de User Mode a Kernel Mode, validación de seguridad y luego el retorno a User Mode.
Caso Arquitectónico: Un servidor de bases de datos que procesa miles de conexiones pequeñas simultáneas está gastando demasiada RAM solo manteniendo los procesos vivos. ¿Cómo optimizas esto lógicamente a nivel de SO?,Cambiar la arquitectura del servidor de un modelo basado en "múltiples procesos" a un modelo basado en "múltiples hilos" (Threads) o asíncrono, ya que los hilos son más ligeros y comparten memoria.
Estás diseñando un SO para un marcapasos cardíaco. ¿Qué tipo de algoritmo de planificación (Scheduler) implementarías?,Un Sistema Operativo de Tiempo Real (RTOS) con un planificador estricto basado en Prioridades, asegurando que los eventos críticos (latidos) siempre preempcionen (interrumpan) cualquier otra tarea.
El Pipelining permite que se terminen de procesar 5 instrucciones en el mismo instante de tiempo,Falso
El Pipelining no procesa varias cosas a la vez en el mismo estado, sino que actúa como una cadena de montaje: mientras una instrucción está en la fase final, otra está a la mitad y otra apenas entra.
Un programa corre bien durante las primeras 3 horas, pero luego vuelve a la computadora inoperable y el disco duro no deja de girar, aunque el programa sigue activo. 
¿Cuál es el diagnóstico lógico,Existe un "Memory Leak" (Fuga de Memoria) severo en el Heap. El SO se quedó sin RAM física e inició el "Thrashing", paginando constantemente hacia el disco duro.
Si dos procesos diferentes cargan la librería estándar "stdio.h" (o similar), el Sistema Operativo carga dos copias idénticas en la memoria RAM física,Falso
 Los SO modernos usan bibliotecas dinámicas compartidas (.dll, .so). Mapean lógicamente la misma ubicación física en la RAM a los espacios virtuales de ambos procesos para ahorrar memoria.
Tienes una matriz de 10,000 x 10,000 en C++. Iterarla por filas toma 1 segundo, iterarla por columnas toma 45 segundos. Sabiendo que los datos son los mismos, ¿a qué componente de la arquitectura se debe esto?,A la Caché y la Localidad Espacial. Los lenguajes como C++ guardan las matrices en filas contiguas. Al leer por filas, aprovechas los bloques enteros subidos a la Caché. Al leer por columnas, causas un "Cache Miss" en cada salto.
El DMA (Direct Memory Access) es utilizado por el teclado para enviar cada pulsación de tecla a la CPU,Falso
 El teclado es de bajo volumen de datos y utiliza Interrupciones. El DMA está reservado para transferencias masivas de datos continuos, como una lectura de un disco duro sólido o una tarjeta de red.
¿Qué es un Analizador Léxico (Lexer o Scanner)?,Es la primera fase del compilador. Lee el código fuente carácter por carácter y agrupa los símbolos en unidades con significado lógico llamadas "Tokens" (ej. identificadores, palabras reservadas).
¿Qué es un Analizador Sintáctico (Parser)?,Es la segunda fase. Toma los tokens del lexer y verifica si su orden es válido según las reglas gramaticales del lenguaje, construyendo un Árbol de Sintaxis Abstracta (AST).
¿Qué es un Árbol de Sintaxis Abstracta (AST)?,Representación de estructura de datos jerárquica (árbol) del código fuente que elimina detalles de formato (como paréntesis) y refleja puramente la lógica matemática y de control.
Diseñas un compilador para un lenguaje científico estricto (como Fortran o C). El código pasa la validación sintáctica, pero al ejecutar lanza un error porque intentaste sumar un string con un int. ¿Qué fase del compilador falló en prevenir esto?,El Analizador Semántico. Es responsable de verificar el significado lógico, como la compatibilidad y el chequeo estricto de tipos de datos.
¿Cuál es la diferencia de arquitectura entre un Compilador y un Intérprete?,El compilador traduce todo el código fuente a lenguaje máquina (binario) de una sola vez creando un ejecutable. El intérprete traduce y ejecuta el código instrucción por instrucción en tiempo de ejecución.
¿Qué es el "Código Intermedio" (Intermediate Representation - IR)?,Un lenguaje de bajo nivel independiente del hardware (como el bytecode de Java). Permite aislar el lexer/parser del generador de código máquina, facilitando que el compilador soporte múltiples arquitecturas de CPU.
¿Qué función tiene la Tabla de Símbolos en el proceso de compilación?,Es una estructura de datos central que almacena metadatos sobre los identificadores (variables, funciones), como su tipo de dato, ubicación en memoria y alcance (scope), siendo consultada por todas las fases.
El proceso de "Enlazado" (Linking) ocurre antes de compilar el código,Falso
 Ocurre al final. El Linker toma los archivos de código objeto generados y los combina con bibliotecas externas estáticas para crear el archivo ejecutable final.
Modelo OSI: ¿Cuál es la función central de la Capa de Enlace de Datos (Capa 2)?,Proporcionar tránsito de datos a través de un enlace físico directo. Utiliza direcciones MAC (físicas) y detecta errores en las tramas (frames) mediante sumas de verificación.
Modelo OSI: ¿Cuál es la función central de la Capa de Red (Capa 3)? A: El Enrutamiento (Routing),Determina la ruta lógica más eficiente para mover paquetes de datos a través de múltiples subredes desde el origen hasta el destino utilizando direcciones IP.
Modelo OSI: ¿Cuál es la diferencia de diseño entre TCP y UDP en la Capa de Transporte (Capa 4)?,TCP establece conexión previa (Handshake) y garantiza que los paquetes lleguen completos y en orden. UDP envía paquetes sin conexión ni garantías, siendo más rápido pero propenso a pérdidas de datos.
¿Cuál es la función de un servidor DNS (Domain Name System)?,Actúa como el directorio de internet, traduciendo nombres de dominio legibles por humanos (ej. google.com) a direcciones IP lógicas enrutables.
El protocolo DHCP puede asignar direcciones MAC dinámicamente a los dispositivos de una red,Falso
 La dirección MAC es un identificador físico e inmutable grabado en la tarjeta de red. DHCP asigna dinámicamente direcciones IP lógicas, máscaras de subred y puertas de enlace.
Un equipo en tu oficina no puede cargar páginas web externas, pero puede enviar paquetes "ping" exitosamente a la impresora local y a otras computadoras en la misma habitación. ¿Qué configuración falta?,Falta o está mal configurado el "Default Gateway" (Puerta de enlace predeterminada), que es la IP del router necesaria para enrutar tráfico fuera de la red local.
Un cliente envía datos bancarios por HTTPS. ¿En qué capa del modelo OSI ocurre el descifrado criptográfico (TLS/SSL) para que la aplicación pueda leer los datos en texto plano?,Tradicionalmente en la Capa de Presentación (Capa 6), responsable del formateo, cifrado y compresión de los datos antes de pasarlos a la Capa de Aplicación.
